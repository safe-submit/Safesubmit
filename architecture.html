ScanCore Data Validation Lifecycle

Detailed step-by-step documentation describing how content flows through ScanCore.

1

Channel Classification

Incoming content is analyzed to determine channel: plaintext, files (txt/md), or images. Classification triggers the correct parsing pipeline and selects encoder models tuned for that channel.

Key considerations:

  • Heuristic headers and mimetype detection
  • User agent and metadata influence
  • Auto-fallback strategies for ambiguous payloads
2

Proxy Serialization

Content is normalized into a canonical proxy object that preserves context and provenance.

Serialization includes:

  • Tokenization and positional offsets
  • Embedded metadata (source, timestamps)
  • Vector embedders per-channel and chunking strategies
3

ScanCore Assessment

The assessment engine runs multi-vector similarity checks, policy evaluation, and severity scoring. Outputs include verdicts, evidence spans and recommended redactions.

Outputs:

  • Structured assessment JSON (status, risk, excerpts)
  • Confidence and rationale traces
  • Integrations for reporting and enforcement
